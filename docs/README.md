## 経歴要約

慶應義塾大学大学院（修士）を卒業後、2020 年 4 月に株式会社エス・エム・エスにデータサイエンティストとして入社しました。

主に介護職専門の求人広告サイトを担当し、サイト内のレコメンド機能の改善など、データ活用を通じた事業貢献を担っていました。
上長のリードのもと、セールスやマーケティング担当者からの課題感ヒアリング、分析・開発仕様の設計、実装・リリース、担当者や役員への成果報告 を一気通貫で行っていました。

データサイエンティストとして働きつつもエンジニアとしてのスキルを身につけたいと思い、2023 年からはアプリケーションエンジニアも兼務しています。
既存プロダクトのフルリニューアルプロジェクトに参画し、周りから大いにサポートされながらインフラ・サーバーサイド・フロントエンドをまんべんなく触っています。

学生時代はオペレーションズ・リサーチ（数理最適化）を専攻しており、修士修了までに異なるテーマで主著論文を 3 本執筆しました。

学生時代のインターンや研究の関係で地理データの扱いに慣れています。今でもたまに副業で地理や GPS データなどの分析を行っています。

個人ではお笑いなど自分の趣味に関するシステム開発やデータ分析を行なっています。

## 職務経歴

### 株式会社エス・エム・エス（2020/04 ～現在）

#### 数理最適化による、訪問介護士のシフト自動作成機能の開発（PoC）

- 担当範囲：プロジェクトの立ち上げ、データ探索、モデルの定式化・実装、結果報告、学会発表

- 介護事業所向けに提供している経営支援サービス『カイポケ』のデータを使い、数理最適化モデルによって訪問介護士のシフトを自動で作成する機能を開発・検証しました。
  詳細は[こちらのブログ](https://tech.bm-sms.co.jp/entry/2023/03/28/120000)や[学会発表で使ったスライド](https://tech.bm-sms.co.jp/entry/2023/10/05/140000)をご参照ください。
  学生時代のスキルを活かせた嬉しさと、数理最適化を実務で使う難しさを感じました。

#### 求人広告サイトにおける求人へのタグ付け

- 担当範囲：課題感のヒアリング、必要なタグや条件の整理、システム設計・実装・リリース、成果報告

- 求人検索をしやすくするために、求人に対して「無資格 OK」「夜勤なし OK」などのタグを、求人内容から正規表現によって自動的に付与する仕組みを開発しました。
  （この施策のみの効果とはいえませんが）サイト内の求人検索回数が増え、検索 → 求人閲覧 → 応募 という経路の応募数も増えました。
  今ではレコメンドや求人の質改善など様々な施策で使われる重要なデータになっています。
  広範囲を担当した初めてのプロジェクトで、設計や実装より関係者との調整に苦労しましたが貴重な経験になりました。

#### ユーザーの行動履歴や求人タグを使ったレコメンドアルゴリズム開発

- 担当範囲：レコメンドアルゴリズムの設計・実装・リリース、成果報告

- 求人広告サイト内のレコメンド機能による応募を増やすために、ユーザーのアクセスログや上記の求人タグを使って複数のレコメンドアルゴリズムを開発しました。
  レコメンドされた求人をクリックしてからの応募率が既存と比べて大幅に増加しました。
  自分で考えたアルゴリズムで成果が出てとても嬉しかったです。

#### クローリングデータを使った営業リストの作成

- 担当範囲：担当者との突合ルールすり合わせ、設計・実装・運用、成果報告

- 運営している人材紹介サイトの新規求人獲得のために、厚労省による介護事業所検索サイトのクローリングデータと自社の事業所データをルールベースで突合し、自社で掲載がない事業所を検出する仕組みを開発しました。

#### （アプリケーションエンジニアとして）[『カイポケ』フルリニューアルプロジェクト](https://careers.bm-sms.co.jp/engineer/kaipoke-renewal)

- 担当範囲：バックエンド（Kotlin, Spring Boot）、フロントエンド（TypeScript, Next.js）、インフラ（AWS）

- バックエンド、フロントエンド、インフラをまんべんなく触るチームにて、アカウントの管理機能や社内のサポートチームが使う機能などを開発しました。
